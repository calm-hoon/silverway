# SilverWay 공모전 시연 시나리오

서비스명: SilverWay — AI 기반 고령자 이동 및 면허 반납 의사결정 지원 서비스

시연 원칙:
- "운전 위험 지수" 표현 사용 (사고 확률 금지)
- "의사결정 보조" 강조
- "과거 패턴 기반 예측형 혼잡도" 표현 사용
- 면허 반납은 가족과 함께 논의할 수 있는 주제로 표현
- 실제 사고 가능성을 단정하지 않음

---

## 시연 흐름 (약 5~7분)

### Step 1. 홈 화면 접속

URL: `https://배포도메인/` 또는 `https://silverway.codegenie.co.kr/`

시연 문구:
> "SilverWay는 AI와 공공데이터를 활용해 고령자와 가족이 이동 방식을 함께 고민할 수 있도록 돕는 서비스입니다. 실제 사고 가능성을 예측하는 것이 아니라, 의사결정 보조 정보를 제공합니다."

확인 포인트:
- 서비스 한 줄 소개: "AI 기반 고령자 이동 및 면허 반납 의사결정 지원 서비스"
- CTA 버튼 "분석 시작하기" 표시

---

### Step 2. 분석 입력 화면 이동

URL: `/analyze`

---

### Step 3. 출발지 검색

입력: "대전광역시청"

시연 문구:
> "출발지에 '대전광역시청'을 검색합니다. Kakao Local API를 통해 실제 주소와 좌표를 가져옵니다."

확인 포인트:
- 검색 결과 드롭다운 표시
- 목록에서 "대전광역시청" 선택
- 선택 후 입력란에 장소명 표시

fallback 상황 설명 (API 키 미설정 시):
> "API 키가 없어도 예시 장소 목록이 표시됩니다. 서비스 안정성을 위한 fallback 설계입니다."

---

### Step 4. 도착지 검색

입력: "충남대학교병원" 또는 "대전역"

확인 포인트:
- 도착지 검색 및 선택

---

### Step 5. 출발 시간 선택

선택: "오늘 오전" 또는 "직접 입력"

시연 문구:
> "출발 시간을 선택하면 해당 시간대의 과거 대중교통 혼잡 패턴이 분석에 반영됩니다."

---

### Step 6. 연령대 선택

선택: "70대"

시연 문구:
> "연령대를 선택하면 운전 위험 지수 산정 시 연령 보정이 적용됩니다."

---

### Step 7. 분석 시작

버튼: "분석 시작하기" 클릭

시연 문구:
> "분석 요청이 전송됩니다. ODsay 대중교통 경로, 기상청 날씨 정보, Claude AI 리포트가 병렬로 조회됩니다."

확인 포인트:
- 로딩 화면 표시 ("분석 결과를 준비하고 있습니다.")
- 완료 후 `/result/[id]` 이동

---

### Step 8. 결과 화면 확인

URL: `/result/[id]`

---

### Step 9. 운전 위험 지수 확인

시연 문구:
> "운전 위험 지수는 공공데이터 기반 사고 패턴, 시간대, 기상 조건, 연령대, 이동 지역 특성을 조합한 의사결정 보조용 점수입니다. 실제 사고 가능성을 의미하지 않습니다."

확인 포인트:
- 점수(0~100)와 레벨(낮음/보통/높음) 표시
- "운전 위험 지수" 레이블 확인

---

### Step 10. 대중교통 대안 확인

시연 문구:
> "ODsay API를 통해 대중교통 경로를 조회합니다. 지하철, 버스 단계별 이동 시간이 표시됩니다."

확인 포인트:
- 총 소요 시간, 환승 횟수, 단계별 이동 수단 표시

---

### Step 11. 과거 패턴 기반 예측형 혼잡도 확인

시연 문구:
> "혼잡도는 실시간 데이터가 아니라 AFC 과거 재차인원 패턴을 기반으로 예측한 값입니다. '과거 패턴 기반 예측형 혼잡도'로 명확히 표기됩니다."

확인 포인트:
- 혼잡도 레벨(여유/보통/혼잡) 표시
- "과거 패턴 기반 예측형 혼잡도" 레이블 확인

---

### Step 12. 기상 요인 확인

시연 문구:
> "기상청 단기예보 API를 통해 출발 시간대 날씨를 조회합니다. 기상 조건이 운전 위험 지수 가중치에 반영됩니다."

확인 포인트:
- 날씨 상태, 기온, 풍속 표시

---

### Step 13. 가족 공유용 리포트 확인

시연 문구:
> "Claude AI가 분석 결과를 바탕으로 가족과 공유할 수 있는 안내 문구를 생성합니다. 면허 반납을 강요하지 않고, 가족과 함께 이동 방식을 논의할 수 있는 내용으로 구성됩니다."

확인 포인트:
- 리포트 제목, 요약, 가족 메시지 표시
- Claude 또는 template fallback으로 생성됨

---

### Step 14. 리포트 복사하기

버튼: "복사하기" 클릭

시연 문구:
> "가족 메시지를 클립보드에 복사해 카카오톡 등으로 공유할 수 있습니다."

---

### Step 15. 지도 확인

시연 문구:
> "Kakao Map JavaScript SDK로 출발지와 도착지 마커를 표시합니다. 지도의 선은 실제 도로 경로가 아니라 두 지점을 연결하는 참고선입니다."

확인 포인트:
- 출발지/도착지 마커 표시
- "실제 이동 경로가 아닙니다" 범례 확인

---

### Step 16. fallback 상황 설명

시연 문구:
> "SilverWay는 외부 API 연결 상태와 관계없이 화면이 깨지지 않도록 설계되어 있습니다. Kakao, ODsay, 기상청, Claude 중 어느 API가 실패해도 예시 데이터(fallback)로 화면이 유지됩니다. 이는 공모전 시연 환경에서도 안정적인 서비스를 보장합니다."

확인 포인트:
- fallback 시 "일부 외부 데이터를 불러오지 못해 예시 데이터를 함께 사용했습니다" 안내 표시
- 화면이 깨지지 않음

---

## 시연 백업 플랜

API 연동이 되지 않는 상황에서도 시연 가능한 URL:

- `/result/test` — sampleAnalysis 기반 예시 결과 (항상 표시됨)
- `/api/health` — 환경변수 설정 여부 확인

시연 중 오류 발생 시:
> "분석 요청 중 외부 데이터 연결이 원활하지 않아 예시 결과를 보여드리겠습니다."
> `/result/test` 페이지로 이동
