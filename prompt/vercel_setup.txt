================================================================
SilverWay — Vercel 배포 가이드
================================================================

이 가이드는 SilverWay 서비스를 Vercel에 처음 배포하는 순서를
설명합니다. GitHub 저장소 연결부터 커스텀 도메인까지 포함합니다.

사전 조건:
  - GitHub 저장소에 최신 코드가 push되어 있어야 합니다.
  - Supabase 프로젝트가 생성되어 있어야 합니다. (supabase_setup.txt 참조)
  - npm run build가 로컬에서 성공해야 합니다.

----------------------------------------------------------------
1단계. Vercel 계정 준비
----------------------------------------------------------------

1. https://vercel.com 접속
2. "Sign Up" 또는 "Log In" → GitHub 계정으로 로그인 (권장)

----------------------------------------------------------------
2단계. 새 프로젝트 생성
----------------------------------------------------------------

1. Vercel 대시보드 → "Add New..." → "Project" 클릭

2. "Import Git Repository" 화면에서:
   - GitHub 계정 연결 (처음이면 GitHub OAuth 승인)
   - 저장소 목록에서 silverway 저장소 찾기
   - "Import" 클릭

3. 프로젝트 설정 화면:

   Framework Preset: Next.js (자동 감지됨)
   Root Directory:   ./  (기본값 유지)
   Build Command:    npm run build  (기본값 유지)
   Install Command:  npm install    (기본값 유지)
   Output Directory: .next          (기본값 유지)

   ※ 설정을 변경하지 않아도 됩니다.

4. 아래 "Environment Variables" 섹션으로 이동하기 전에
   "Deploy"를 누르지 마세요. 환경변수를 먼저 설정합니다.

----------------------------------------------------------------
3단계. 환경변수 설정 (핵심)
----------------------------------------------------------------

"Configure Project" 화면의 "Environment Variables" 섹션에서
아래 8개를 모두 입력합니다.

입력 방법:
  - Name 칸에 변수명 입력
  - Value 칸에 실제 값 입력
  - Environment: Production, Preview, Development 모두 체크
  - "Add" 클릭
  - 8개 모두 반복

┌─────────────────────────────────────────────────────────────┐
│ 브라우저 노출 가능 (NEXT_PUBLIC_ 접두사)                    │
├────────────────────────────┬────────────────────────────────┤
│ 변수명                     │ 값                             │
├────────────────────────────┼────────────────────────────────┤
│ NEXT_PUBLIC_SUPABASE_URL   │ https://<id>.supabase.co       │
│ NEXT_PUBLIC_SUPABASE_ANON_KEY │ eyJhbGciOiJI...            │
│ NEXT_PUBLIC_KAKAO_MAP_KEY  │ 카카오 JavaScript SDK 앱 키    │
└────────────────────────────┴────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ 서버 전용 (NEXT_PUBLIC_ 접두사 없음 — 절대 혼용 금지)       │
├────────────────────────────┬────────────────────────────────┤
│ 변수명                     │ 값                             │
├────────────────────────────┼────────────────────────────────┤
│ SUPABASE_SERVICE_ROLE_KEY  │ eyJhbGciOiJI... (service role) │
│ KAKAO_REST_API_KEY         │ 카카오 REST API 키             │
│ ODSAY_API_KEY              │ ODsay 대중교통 API 키          │
│ WEATHER_API_KEY            │ 공공데이터포털 기상청 serviceKey│
│ ANTHROPIC_API_KEY          │ Claude API 키                  │
└────────────────────────────┴────────────────────────────────┘

각 키 발급처:
  - NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY,
    SUPABASE_SERVICE_ROLE_KEY
    → Supabase 대시보드 > Project Settings > API

  - NEXT_PUBLIC_KAKAO_MAP_KEY (JavaScript 키)
  - KAKAO_REST_API_KEY (REST API 키)
    → https://developers.kakao.com
      내 애플리케이션 > silverway 앱 > 앱 키
      두 키는 같은 앱에서 발급하지만 종류가 다릅니다.

  - ODSAY_API_KEY
    → https://lab.odsay.com
      대중교통 길찾기 API 신청 후 발급

  - WEATHER_API_KEY
    → https://www.data.go.kr
      "기상청_단기예보 조회서비스" 신청
      승인 후 발급되는 일반 인증키(serviceKey) 사용

  - ANTHROPIC_API_KEY
    → https://console.anthropic.com
      API Keys 메뉴에서 발급

주의:
  - SUPABASE_SERVICE_ROLE_KEY를 NEXT_PUBLIC_으로 설정하면
    브라우저에 노출되어 보안 사고가 발생합니다.
  - KAKAO_REST_API_KEY도 서버 전용입니다.
  - 값에 앞뒤 공백이 들어가지 않도록 주의하세요.

키가 없어도 배포 가능:
  - 키가 없으면 해당 API는 fallback 응답을 반환합니다.
  - 화면이 깨지지 않습니다.
  - 단, 실제 API 연동이 필요한 시연을 위해서는 키 설정 필수.

----------------------------------------------------------------
4단계. 배포 실행
----------------------------------------------------------------

환경변수 입력 완료 후:

1. "Deploy" 버튼 클릭
2. 빌드 로그 화면에서 진행 상황 확인 (약 1~3분 소요)
3. "Congratulations!" 또는 초록색 체크 → 배포 성공

배포 실패 시:
  - Build Logs 탭에서 오류 메시지 확인
  - TypeScript 오류: 로컬에서 npm run build 먼저 성공시키기
  - 환경변수 오류: 변수명 오타 확인

----------------------------------------------------------------
5단계. 배포 후 동작 확인
----------------------------------------------------------------

Vercel 대시보드에서 배포된 도메인 확인:
  예: https://silverway-xxxx.vercel.app

아래 순서로 확인합니다.

1. 헬스체크 (환경변수 설정 여부 확인)
   URL: https://silverway-xxxx.vercel.app/api/health

   curl https://silverway-xxxx.vercel.app/api/health

   정상 응답:
   {
     "ok": true,
     "service": "SilverWay",
     "status": "ready",
     "environment": "production",
     "checks": {
       "supabaseUrl": true,
       "supabaseAnonKey": true,
       "supabaseServiceRoleKey": true,
       "kakaoMapKey": true,
       "kakaoRestApiKey": true,
       "odsayApiKey": true,
       "weatherApiKey": true,
       "anthropicApiKey": true
     }
   }

   checks 항목이 false이면 해당 환경변수가 미설정된 것입니다.

2. 주요 페이지 접속 확인
   - https://silverway-xxxx.vercel.app/
   - https://silverway-xxxx.vercel.app/analyze
   - https://silverway-xxxx.vercel.app/result/test

3. 분석 API 확인
   curl -X POST https://silverway-xxxx.vercel.app/api/analyze \
     -H "Content-Type: application/json" \
     -d '{
       "origin": {
         "name": "대전광역시청",
         "address": "대전 서구 둔산로 100",
         "lat": 36.3504,
         "lng": 127.3845
       },
       "destination": {
         "name": "충남대학교병원",
         "address": "대전 중구 문화로 282",
         "lat": 36.3166,
         "lng": 127.4156
       },
       "departureTime": "2026-05-04T10:00:00+09:00",
       "ageGroup": "70s"
     }'

   응답에서 resultId 확인 후:
   curl https://silverway-xxxx.vercel.app/api/result/<resultId>

----------------------------------------------------------------
6단계. 커스텀 도메인 설정 (silverway.codegenie.co.kr)
----------------------------------------------------------------

Vercel 대시보드 → 프로젝트 선택 → Settings → Domains

1. 도메인 입력 칸에 입력:
     silverway.codegenie.co.kr
   → "Add" 클릭

2. Vercel이 DNS 설정 안내를 표시합니다:
   아래 두 가지 중 하나를 선택합니다.

   방법 A — CNAME 레코드 (서브도메인에 권장):
     Type:   CNAME
     Name:   silverway
     Value:  cname.vercel-dns.com

   방법 B — A 레코드 (apex 도메인인 경우):
     Type:   A
     Name:   @
     Value:  76.76.21.21  (Vercel이 안내하는 IP)

3. DNS 설정 위치 (codegenie.co.kr 관리 업체에서 설정):
   - 도메인 관리 업체(가비아, 카페24, Route53 등) 접속
   - DNS 관리 메뉴에서 위 레코드 추가

4. DNS 전파 대기:
   - 보통 수 분 ~ 최대 48시간 소요
   - Vercel 대시보드에서 "Valid Configuration" 표시 확인

5. HTTPS 자동 적용 확인:
   - Vercel이 Let's Encrypt SSL 인증서를 자동 발급합니다.
   - 도메인 연결 후 자동으로 HTTPS가 적용됩니다.
   - https://silverway.codegenie.co.kr 접속 확인

----------------------------------------------------------------
7단계. 이후 재배포 방법
----------------------------------------------------------------

코드 변경 후 GitHub에 push하면 자동으로 재배포됩니다.

  git add .
  git commit -m "update"
  git push origin master

Vercel 대시보드 → Deployments 탭에서 진행 상황 확인 가능.

수동 재배포:
  Vercel 대시보드 → Deployments → 최근 배포 → "..." → "Redeploy"

환경변수 변경 후 재배포:
  Settings → Environment Variables → 수정 → Save
  이후 반드시 수동 재배포 필요 (환경변수는 새 배포부터 적용됨)

----------------------------------------------------------------
8단계. Vercel 함수 실행 시간 설정 (선택)
----------------------------------------------------------------

Claude API 호출이 15초 타임아웃으로 설정되어 있으므로
Vercel의 기본 함수 실행 제한(10초)과 충돌할 수 있습니다.

vercel.json이 없는 경우 기본값(10초)이 적용됩니다.
Claude API fallback이 즉시 작동하므로 서비스는 정상 유지되지만
실제 Claude 응답을 원한다면 아래 설정을 추가할 수 있습니다.

프로젝트 루트에 vercel.json 생성 (선택):

  {
    "functions": {
      "app/api/report/route.ts": {
        "maxDuration": 30
      },
      "app/api/analyze/route.ts": {
        "maxDuration": 30
      }
    }
  }

주의:
  - Free 플랜: 최대 10초
  - Pro 플랜: 최대 300초
  - 현재 fallback 구조 덕분에 타임아웃 발생 시 template 리포트를 반환합니다.
