너는 15년차 Next.js 개발자이자 공공데이터·AI 서비스 운영 QA/보안 점검 전문가다.
나는 2026년 대전광역시 공공데이터·AI 활용 창업경진대회에 제출할 SilverWay 서비스를 개발하고 있다.

서비스명은 SilverWay이고, 한 줄 정의는 “AI 기반 고령자 이동 및 면허 반납 의사결정 지원 서비스”다.

현재 상황:
- 이전 작업 1~24는 완료되었다.
- 실사용 전환 프롬프트 1~7도 진행했거나 진행 중이다.
- Mock/시연용 표현 정리, RecentPlaceList localStorage, Kakao Local 디버깅, ODsay 디버깅, Supabase 저장/조회 확인, Claude 운영 확인, 운영 Smoke Test까지 진행했다.
- 이제는 새 기능 구현이 아니라 최종 제출/운영 전 표현 원칙과 보안 위험을 점검해야 한다.

이번 작업은 새롭게 시작하는 실사용 전환 프롬프트 8번이다.
파일명 기준으로는 8.txt 작업이다.

이번 작업명:
표현·보안 최종 QA

작업 목적:
- 사용자 화면, API 응답, 문서, 테스트, 코드 주석에서 위험한 표현이나 운영에 부적절한 표현을 찾는다.
- API 키, service role key, 환경변수, localStorage, console.log, API 응답의 보안 위험을 점검한다.
- 실제 서비스 기준으로 “노출되면 안 되는 것”과 “사용자에게 오해를 줄 수 있는 표현”을 제거하거나 수정한다.
- 이번 작업은 QA 중심이며, 필요한 경우 작은 문구 수정과 보안 노출 제거만 허용한다.
- 기능 구현, 디자인 변경, 대규모 리팩토링은 하지 않는다.

절대 하지 말 것:
- 새 기능 구현하지 마.
- 디자인 변경하지 마.
- 레이아웃 변경하지 마.

- Kakao/ODsay/Weather/Claude 연동 로직을 재작성하지 마.

- fallback 구조를 제거하지 마.
- 회원가입/auth 구현하지 마.
- 관리자 페이지 구현하지 마.
- 결제/알림/카카오 공유 구현하지 마.
- 패키지 설치하지 마.
- 대규모 리팩토링하지 마.
- 테스트를 삭제하지 마.
- 금지 표현을 새로 추가하지 마.

서비스 표현 원칙:
- 위험도 점수는 실제 사고 확률이 아니라 “운전 위험 지수”다.
- “사고 확률”, “예측 확률”, “probability” 표현은 사용자 화면/API 응답/운영 문서에 노출하지 않는다.
- AFC 혼잡도는 실시간이 아니라 “과거 패턴 기반 예측형 혼잡도”로 표현한다.
- fallback은 운영 안정성을 위한 대체 응답으로 유지한다.
- 면허 반납은 강요가 아니라 가족과 함께 논의할 수 있는 의사결정 보조 관점으로 표현한다.
- “반드시 반납”, “운전 금지”처럼 강제적·공포 조장 표현은 사용하지 않는다.

1. 표현 원칙 QA

아래 표현을 검색한다.

금지 또는 수정 대상:
- 사고 확률
- 예측 확률
- probability
- accidentProbability
- predictionProbability
- 실시간 혼잡도
- live congestion
- realtime congestion
- 반드시 반납
- 운전 금지

- 시연용
- Mock
- MOCK
- mock
- 더미
- demo
- sample result
- test result

허용 가능 예외:
- “실제 사고 확률이 아니라”처럼 부정 문맥으로 위험도 정의를 설명하는 기존 핵심 문구는 허용 가능하다.
  단, 사용자 화면에서 과도하게 반복되면 더 부드러운 표현으로 개선 후보에 올린다.
- 테스트 파일에서 과거 mode 값 변경을 검증하기 위한 문자열은 허용 가능하다.
- lib/fallback, fallbackFlags 등 운영 안정성 용어는 허용 가능하다.
- 파일명 sampleAnalysis가 기존 호환성 때문에 남아 있는 경우는 허용 가능하나, 사용자 화면에 sample이라고 노출되면 수정한다.

권장 교체:
- 사고 확률 / 예측 확률 → 운전 위험 지수
- 실시간 혼잡도 → 과거 패턴 기반 예측형 혼잡도
- 반드시 반납 → 가족과 함께 논의해볼 수 있습니다
- 운전 금지 → 대중교통 또는 동행 이동도 함께 검토해보세요
- Mock / 시연용 / 더미 → 예시, 보조 데이터, 대체 응답, fallback 기반 응답
- sample result → 예시 결과 또는 참고용 결과

4. 보안 QA

검색할 키워드:
- SUPABASE_SERVICE_ROLE_KEY
- ANTHROPIC_API_KEY
- KAKAO_REST_API_KEY
- ODSAY_API_KEY
- WEATHER_API_KEY
- NEXT_PUBLIC_ODSAY
- NEXT_PUBLIC_WEATHER
- NEXT_PUBLIC_ANTHROPIC
- NEXT_PUBLIC_KAKAO_REST
- console.log
- console.error
- localStorage
- sessionStorage
- Authorization
- apiKey

확인 기준:
- SUPABASE_SERVICE_ROLE_KEY는 서버 전용 파일에서만 사용되어야 한다.
- ANTHROPIC_API_KEY는 서버 전용 파일에서만 사용되어야 한다.
- KAKAO_REST_API_KEY는 서버 전용 파일에서만 사용되어야 한다.
- ODSAY_API_KEY는 서버 전용 파일에서만 사용되어야 한다.
- WEATHER_API_KEY는 서버 전용 파일에서만 사용되어야 한다.
- NEXT_PUBLIC으로 허용되는 것은 다음 정도만이다.
  - NEXT_PUBLIC_SUPABASE_URL
  - NEXT_PUBLIC_SUPABASE_ANON_KEY
  - NEXT_PUBLIC_KAKAO_MAP_KEY
- localStorage/sessionStorage에 API 키, service role key, Claude 응답 원본 전체, 내부 에러 전문, 민감정보가 저장되면 안 된다.
- console.log에 API 키나 전체 요청 URL(apiKey 포함)이 출력되면 안 된다.
- console.error는 서버 로그에서 원인 확인용으로 허용하되 민감정보는 마스킹되어야 한다.
- Authorization header 또는 apiKey 포함 URL을 응답에 포함하면 안 된다.
