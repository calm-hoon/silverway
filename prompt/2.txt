서비스명은 SilverWay이고, 한 줄 정의는 “AI 기반 고령자 이동 및 면허 반납 의사결정 지원 서비스”다.

이번 작업명:
RecentPlaceList localStorage 실제 동작화

작업 목적:
- 현재 RecentPlaceList가 samplePlaces 하드코딩 기반이라면, 이를 localStorage 기반 실제 최근 장소/최근 경로 기능으로 전환한다.
- 사용자가 Kakao Local 검색 결과에서 출발지 또는 도착지를 선택하면 해당 장소를 최근 장소로 저장한다.
- 출발지와 도착지를 모두 선택한 뒤 분석을 실행하면 해당 조합을 최근 경로로 저장한다.
- 로그인 없이 브라우저 단위로 동작하게 한다.
- API 키, 환경변수, 민감정보는 절대 localStorage에 저장하지 않는다.
- 기존 디자인, 레이아웃, 색상, 카드 스타일, 버튼 스타일은 변경하지 않는다.

서비스 표현 원칙:
- 위험도 점수는 실제 사고 확률이 아니라 “운전 위험 지수”다.
- “사고 확률”, “예측 확률”, “probability” 표현은 새로 추가하지 않는다.
- AFC 혼잡도는 실시간이 아니라 “과거 패턴 기반 예측형 혼잡도”로 표현한다.

- 면허 반납은 강요가 아니라 가족과 함께 논의할 수 있는 의사결정 보조 관점으로 표현한다.

구현 지침:

1. localStorage 전용 유틸 작성
lib/storage/recentPlaces.ts 파일을 새로 만들거나 기존 파일이 있다면 보강한다.

권장 storage key:
const RECENT_PLACES_KEY = "silverway:recent-places";
const RECENT_ROUTES_KEY = "silverway:recent-routes";

권장 제한:
const MAX_RECENT_PLACES = 10;
const MAX_RECENT_ROUTES = 5;

권장 함수:
export function getRecentPlaces(): RecentPlace[];
export function saveRecentPlace(place: Place): RecentPlace[];
export function removeRecentPlace(id: string): RecentPlace[];
export function clearRecentPlaces(): void;

export function getRecentRoutes(): RecentRoute[];
export function saveRecentRoute(input: { origin: Place; destination: Place }): RecentRoute[];
export function removeRecentRoute(id: string): RecentRoute[];
export function clearRecentRoutes(): void;

주의:
- SSR 환경에서 window가 없으면 빈 배열 반환 또는 no-op 처리한다.
- localStorage JSON parse 실패 시 throw하지 않고 빈 배열 반환한다.
- 잘못된 데이터 구조는 무시한다.
- 저장 시 최신순으로 정렬한다.
- API 키, 환경변수, 내부 에러, 사용자 민감정보는 저장하지 않는다.
- 저장 대상은 장소명, 주소, lat, lng, 마지막 사용 시각 정도로 제한한다.

2. 타입 보강
types/index.ts에 필요한 타입이 없다면 최소 보강한다.

권장 타입:
export type RecentPlace = Place & {
  id: string;
  lastUsedAt: string;
};

export type RecentRoute = {
  id: string;
  origin: Place;
  destination: Place;
  lastUsedAt: string;
};

주의:
- 기존 Place 타입과 충돌하지 않게 한다.
- 기존 AnalysisRequest 타입을 깨지 않게 한다.
- id는 중복 제거가 가능한 안정적 값으로 생성한다.

3. 중복 제거 기준
최근 장소:
- address가 같으면 같은 장소로 본다.
- address가 없으면 name + lat + lng 조합을 사용한다.
- 같은 장소를 다시 저장하면 중복 추가하지 않고 최신순으로 이동한다.

최근 경로:
- origin.address + destination.address가 같으면 같은 경로로 본다.
- address가 없으면 origin 좌표 + destination 좌표 조합을 사용한다.
- 같은 경로를 다시 저장하면 중복 추가하지 않고 최신순으로 이동한다.

4. RecentPlaceList 실제 데이터 연결
components/analyze/RecentPlaceList.tsx에서 samplePlaces 하드코딩이 있다면 제거하거나 fallback용 기본 추천 목록으로만 제한한다.

권장 동작:
- localStorage에 최근 장소가 있으면 최근 장소 목록을 보여준다.
- 최근 장소가 없으면 기존 추천 장소 목록을 보여줄 수 있다.
- 단, 이 경우 사용자에게 “시연용” 또는 “Mock”이라고 표현하지 않는다.
- 제목은 “최근 장소” 또는 “추천 장소”로 유지한다.
- 기존 디자인은 유지한다.
- 카드 모양, 여백, 색상, 아이콘, 버튼 스타일은 바꾸지 않는다.
- 사용자가 장소를 클릭하면 기존 출발지/도착지 선택 흐름에 연결한다.

5. 장소 선택 시 저장
Kakao Local 검색 결과에서 장소를 선택하는 부분을 찾아 saveRecentPlace(place)를 호출한다.

주의:
- 저장은 client component에서만 수행한다.
- 서버 컴포넌트에서 localStorage 접근하지 않는다.
- 검색 결과의 name, address, lat, lng가 모두 유효한 경우에만 저장한다.
- lat/lng가 없으면 저장하지 않거나 안전하게 무시한다.
- 저장 실패가 분석 흐름을 막지 않게 한다.

6. 분석 실행 시 최근 경로 저장
분석 버튼 클릭 로직에서 origin과 destination이 모두 유효할 때 saveRecentRoute({ origin, destination })를 호출한다.

주의:
- 분석 API 요청 전에 저장해도 되고, 요청 성공 후 저장해도 된다.
- 사용자가 분석을 누른 경로가 실제 최근 경로로 남는 것이 중요하다.
- 저장 실패가 /api/analyze 호출을 막지 않게 한다.
- API 키, resultId, 분석 결과 전체 JSON은 최근 경로 localStorage에 저장하지 않는다.

7. UI 상태 처리
- 최근 장소/최근 경로 목록 로딩은 useEffect 이후에 처리한다.
- hydration mismatch가 나지 않도록 초기값은 빈 배열로 두고 클라이언트 마운트 후 읽는다.
- localStorage 접근 불가 환경에서도 화면이 깨지지 않게 한다.
- 최근 장소가 없을 때의 빈 상태 문구를 짧게 유지한다.
- 기존 디자인과 동일한 스타일을 사용한다.

tests/storage/recentPlaces.test.ts를 작성하거나 기존 테스트를 보강한다.

필수 테스트:

- 깨진 JSON이 localStorage에 있어도 throw하지 않는다.
- clearRecentPlaces, clearRecentRoutes가 동작한다.

주의:
