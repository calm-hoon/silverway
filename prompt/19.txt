너는 15년차 Next.js 개발자이자 공공데이터·AI 서비스 MVP 설계 전문가이며, AI API 연동과 fallback 설계에 익숙한 풀스택 개발자다.
나는 2026년 대전광역시 공공데이터·AI 활용 창업경진대회에 제출할 SilverWay 서비스를 개발하고 있다.

서비스명은 SilverWay이고, 한 줄 정의는 “AI 기반 고령자 이동 및 면허 반납 의사결정 지원 서비스”다.
목표는 공모전 시연용 MVP를 넘어 실제 배포 가능한 앱으로 확장 가능한 구조를 만드는 것이다.

중요:
- 이번 작업은 디자인 변경/벤치마킹 반영 단계가 아니다.
- 기존 UI 디자인은 가능한 유지한다.
- 이번 작업은 Claude API를 서버 측에서 호출해 자연어 리포트를 생성하는 단계다.
- Claude API 실패, 키 누락, 응답 오류, 제한 시간 초과 시 반드시 generateTemplateReport fallback을 사용한다.
- 아직 Kakao Local API, Kakao Map 연동은 하지 않는다.
- ODsay 연동 흐름은 유지한다.
- Weather 연동 흐름은 유지한다.
- Supabase 저장/조회 흐름은 유지한다.
- 아직 회원가입/auth 구현하지 않는다.
- 아직 관리자 페이지, 결제, 알림, 카카오 공유 구현하지 않는다.
- ANTHROPIC_API_KEY는 서버 Route Handler에서만 사용하고 클라이언트에 노출하지 않는다.

디자인 관련 주의:
- 이번 작업은 디자인 변경/반영 단계가 아니다.
- 디자인이나 벤치마킹 사이트 반영이 필요한 작업이 다시 오면, 반드시 작업 시작 전에 아래처럼 큰 제목으로 먼저 알려줘.

# 🚨 디자인 변경/벤치마킹 반영 단계입니다

서비스 표현 원칙:
- 위험도 점수는 실제 사고 확률이 아니라 “운전 위험 지수”다.
- “사고 확률”, “예측 확률”, “probability” 표현은 사용하지 않는다.
- AFC 혼잡도는 실시간이 아니라 “과거 패턴 기반 예측형 혼잡도”로 표현한다.
- API 실패 시 sampleAnalysis, sampleRoute, sampleWeather, generateTemplateReport 등 fallback으로 화면이 깨지지 않게 한다.
- “무장애”보다 “이동권 보장”, “이동 편의 동선” 표현을 우선 사용한다.
- 면허 반납은 강요가 아니라 가족과 함께 논의할 수 있는 의사결정 보조 관점으로 표현한다.
- Claude가 생성하는 문장에도 위 표현 원칙이 반드시 적용되어야 한다.

작업 목표:
- Claude API를 서버 측에서 호출하는 리포트 생성 구조를 만든다.
- AnalysisResult 또는 핵심 분석 데이터를 기반으로 가족 공유용 자연어 리포트를 생성한다.
- Claude 응답을 앱 내부 ReportContent 타입으로 안전하게 정규화한다.
- /api/report Route Handler에서 Claude 리포트 생성을 시도하고, 실패 시 generateTemplateReport fallback을 반환한다.
- /api/analyze에서 가능한 경우 Claude 리포트를 반영하고, 실패 시 템플릿 리포트를 유지한다.
- Claude 응답에 금지 표현이 포함되면 템플릿 fallback 또는 안전한 문구 보정 처리를 적용한다.
- API key 누락, 네트워크 실패, 응답 오류, 안전하지 않은 문구 생성 상황에서도 앱이 깨지지 않게 한다.

1. 환경변수
- .env.local.example에 ANTHROPIC_API_KEY가 이미 있는지 확인한다.
- 없으면 추가한다.

ANTHROPIC_API_KEY=

주의:

- 클라이언트 컴포넌트에서 사용하지 않는다.
- 서버 Route Handler 또는 서버 유틸에서만 읽는다.
- 응답 JSON이나 console.log에 key를 노출하지 않는다.

2. lib/report/reportSafety.ts
Claude 응답과 템플릿 리포트에 공통 적용할 안전 검사 유틸을 만든다.

권장 export:
export const FORBIDDEN_REPORT_TERMS = [
  "사고 확률",
  "예측 확률",
  "probability",
  "accident probability",
  "prediction probability",
  "실시간 혼잡도",
  "realtime congestion",
  "live congestion",
  "운전 금지",
  "반드시 반납"
];

export function containsForbiddenReportTerms(text: string): boolean
export function sanitizeReportText(text: string): string
export function validateReportContent(report: ReportContent): { ok: true } | { ok: false; reason: string }

처리 원칙:
- 금지 표현이 포함되면 ok false로 판단한다.
- 간단히 치환 가능한 표현은 sanitizeReportText에서 보정해도 된다.
- 단, 위험한 문구가 많으면 Claude 결과를 버리고 generateTemplateReport fallback을 사용한다.
- “운전 위험 지수”, “의사결정 보조”, “과거 패턴 기반 예측형 혼잡도” 표현은 유지한다.

3. lib/report/normalizeClaudeReport.ts
Claude 응답 텍스트를 ReportContent로 변환한다.
Claude에게 JSON 응답을 요청하더라도 실제 응답이 깨질 수 있으므로 안전하게 처리한다.

권장 함수:
export function normalizeClaudeReport(rawText: string): ReportContent | null

처리 기준:
- JSON 파싱이 되면 title, body, familyMessage를 추출한다.
- JSON 파싱이 안 되면 raw text를 body로 사용하되 title/familyMessage를 안전하게 만든다.
- 필수 필드가 비어 있으면 null을 반환한다.
- 금지 표현이 포함되면 null을 반환하거나 sanitize 후 validate한다.
- 너무 긴 familyMessage는 적당히 줄인다.
- 면허 반납 강요 문구가 있으면 null 처리한다.

4. lib/report/generateClaudeReport.ts
Claude API 호출 함수와 fallback 구조를 만든다.

권장 타입:
export type ClaudeReportInput = {
  analysis: AnalysisResult;
};

export type ClaudeReportResult =
  | { ok: true; report: ReportContent; source: "CLAUDE" }
  | { ok: false; report: ReportContent; source: "TEMPLATE"; reason: string };

권장 함수:
export async function generateClaudeReport(input: ClaudeReportInput): Promise<ClaudeReportResult>

처리 흐름:
1. ANTHROPIC_API_KEY 확인
2. 키가 없으면 generateTemplateReport fallback 반환
3. AnalysisResult에서 필요한 정보만 추출
4. Claude에 보낼 prompt 구성
5. timeout 또는 AbortController 적용
6. Claude 응답 수신
7. normalizeClaudeReport 적용
8. validateReportContent 적용
9. 성공 시 source "CLAUDE"
10. 실패 시 generateTemplateReport fallback

Claude 프롬프트 원칙:
- 한국어로 작성한다.
- 결과는 가능하면 JSON으로 요청한다.
- title, body, familyMessage를 포함하도록 요청한다.
- 절대 사용하면 안 되는 표현을 명시한다.
- 위험도는 “운전 위험 지수”로만 표현하게 한다.
- 혼잡도는 “과거 패턴 기반 예측형 혼잡도”로만 표현하게 한다.
- 면허 반납은 강요하지 않고 가족 논의 주제로만 표현하게 한다.
- 공포감을 주는 문장보다 차분한 안내 문장을 사용하게 한다.

Claude에 전달할 데이터:
- 출발지/도착지
- 출발 시간
- 연령대
- drivingRisk.score, level, factors
- transit.durationMin, transferCount, steps 요약
- congestion level, description
- weather condition, riskScore, description
- dataSources
- fallbackFlags

주의:
- 개인정보나 민감정보는 전달하지 않는다.
- API key를 prompt에 포함하지 않는다.
- Claude 응답을 그대로 신뢰하지 말고 안전 검사 후 사용한다.
- 실패를 throw로 전파하지 않고 fallback으로 반환한다.

6. app/api/report/route.ts
- 기존 템플릿 리포트 route를 Claude 연동 route로 확장한다.
- POST body로 AnalysisResult 또는 관련 데이터를 받는다.
- Claude 생성 성공 시 source "CLAUDE"로 반환한다.
- 실패 시 generateTemplateReport fallback 결과를 source "TEMPLATE"로 반환한다.
- 실패해도 200 fallback 응답을 우선한다.

권장 응답 구조:
{
  "ok": true,
  "mode": "CLAUDE_OR_TEMPLATE",
  "data": ReportContent,
  "meta": {
    "source": "CLAUDE",
    "fallback": false
  }
}

fallback 응답 예시:
{
  "ok": true,
  "mode": "CLAUDE_OR_TEMPLATE",
  "data": ReportContent,
  "meta": {
    "source": "TEMPLATE",
    "fallback": true,
    "reason": "Anthropic API key is missing"
  }
}

주의:
- ANTHROPIC_API_KEY를 응답에 포함하지 않는다.
- Claude 실패 시 500보다 200 fallback을 우선한다.
- 클라이언트에서 Claude API를 직접 호출하지 않는다.

7. app/api/analyze/route.ts
- 기존 Mock 분석, ODsay, Weather, Supabase 저장 흐름을 크게 바꾸지 않는다.
- AnalysisResult 생성 후 generateClaudeReport를 호출해 report를 반영한다.
- Claude 실패 시 generateTemplateReport fallback을 유지한다.
- fallbackFlags.report를 source에 따라 업데이트한다.
- report generatedBy 또는 source 필드가 있다면 "CLAUDE" 또는 "TEMPLATE"로 반영한다.
- Supabase 저장은 최종 report가 반영된 AnalysisResult를 저장하도록 한다.
- Kakao Local, Kakao Map은 아직 호출하지 않는다.
