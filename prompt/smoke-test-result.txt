# SilverWay 운영 Smoke Test 결과

## 1. 테스트 환경
- 테스트 일시: 2026-05-05 22:30 (KST)
- 테스트 URL: https://silverway.codegenie.co.kr
- 브랜치/배포 버전: 로컬 master 기준 1~6.txt 수정사항 미배포 상태 (프로덕션은 이전 버전 실행 중)
- 테스트한 브라우저: 브라우저 직접 접속 미확인 (API curl 테스트 진행)
- 테스트한 기기: Windows 11 / curl

---

## 2. 전체 판단
- 전체 흐름 통과 여부: PARTIAL PASS — 핵심 저장/조회 흐름은 동작하나 Kakao Local·ODsay가 fallback 발생 중
- 실제 API 성공 항목: Health Check, Weather (KMA 실데이터), Claude Report (실 Claude 응답), Supabase 저장, Supabase 조회
- fallback 발생 항목: Kakao Local (GET/POST 모두), ODsay 대중교통 경로
- 즉시 수정 필요 항목:
  1. 로컬 1~6.txt 수정사항을 Vercel에 배포 (MOCK 표현 제거, reason 코드 정정 등)
  2. Kakao Local API 미동작 원인 조사 (GET/POST 모두 SAMPLE 반환 중)
  3. Supabase 저장 시 한국어 origin/destination 이름 인코딩 깨짐 (garbled text)

## 5. 발견된 문제
| 우선순위 | 문제 | 근거 | 다음 조치 |
|---|---|---|---|
| P0 | 로컬 1~6.txt 수정사항 미배포 | mode="MOCK_WITH_STORAGE", message에 "Mock", persistence.saved 없음, meta.reason 없음 | git push → Vercel 배포 |
| P1 | Kakao Local API GET/POST 모두 FALLBACK | GET: reason="검색어가 너무 짧습니다." (7자 쿼리), POST: reason="검색 결과가 없습니다." — KAKAO_REST_API_KEY 존재하나 API 미동작 | Vercel Function Logs 확인, Kakao API 허용 도메인·키 상태 점검 |
| P1 | ODsay reason 빈 문자열 | reason="ODsay error code= msg=" — 4.txt 수정사항(extractOdsayError) 미배포 | 배포 후 재확인 |
| P2 | Supabase 저장 시 한국어 이름 인코딩 깨짐 | origin_name="?????" 형태로 저장, 조회 시 동일하게 깨진 문자 반환 | DB charset 또는 insert 시 인코딩 설정 점검 |
| P2 | weather 필드 DB 조회 시 항상 sampleAnalysis.weather 반환 | rowToAnalysisResult에서 weather: sampleAnalysis.weather 하드코딩, 저장된 실제 KMA 데이터를 복원 못함 | analysis_logs에 weather_summary 컬럼 추가 또는 허용 범위로 정리 |
| P3 | /api/result/[없는id]에 meta.reason 필드 없음 | 5.txt 수정사항(reason: result.reason ?? "RESULT_NOT_FOUND") 미배포 | 배포 후 자동 해결 |
