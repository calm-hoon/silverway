너는 15년차 Next.js 개발자이자 공공데이터·AI 서비스 실서비스 전환 전문가다.
나는 2026년 대전광역시 공공데이터·AI 활용 창업경진대회에 제출할 SilverWay 서비스를 개발하고 있다.

서비스명은 SilverWay이고, 한 줄 정의는 “AI 기반 고령자 이동 및 면허 반납 의사결정 지원 서비스”다.

현재 상황:
- 이전 작업 1~24는 완료되었다.
- Claude 리포트 생성 연동 구조는 이미 구현되어 있다.
- lib/report/generateClaudeReport.ts, lib/report/generateTemplateReport.ts, app/api/report/route.ts 또는 유사 파일이 존재한다.
- fallback 구조, FallbackFlags, 에러/로딩 처리도 이미 구현되어 있다.
- work.txt 점검 결과 기준으로 ANTHROPIC_API_KEY가 Vercel에 없거나 운영 환경에서 false로 보일 수 있으며, 이 경우 Claude 리포트가 항상 템플릿 fallback으로 동작한다.
- 이번 작업은 Claude 연동을 새로 만드는 것이 아니라, 운영 환경에서 Claude API가 실제로 동작하는지 확인하고 필요한 경우 최소 수정하는 작업이다.

이번 작업은 새롭게 시작하는 실사용 전환 프롬프트 6번이다.
파일명 기준으로는 6.txt 작업이다.

이번 작업명:
Claude 운영 확인 및 리포트 fallback 유지 점검

작업 목적:
- ANTHROPIC_API_KEY가 Vercel 운영 환경에서 정상 인식되는지 확인한다.
- /api/report 또는 /api/analyze에서 Claude 리포트가 실제로 호출되는지 확인한다.
- Claude 호출 실패 시 generateTemplateReport fallback이 안전하게 동작하는지 확인한다.
- 실제 Claude 성공/실패 상태를 응답 meta 또는 fallbackFlags로 확인할 수 있게 한다.
- Claude 연동 구조를 재작성하지 않고 최소 수정만 한다.
- 기존 디자인, 레이아웃, UI 스타일은 변경하지 않는다.

절대 하지 말 것:
- Claude 연동을 처음부터 다시 만들지 마.
- Anthropic SDK를 새로 설치하지 마. 이미 fetch 기반 또는 기존 SDK 기반 구현이 있으면 그대로 활용한다.
- generateTemplateReport fallback을 제거하지 마.
- Kakao Local 연동을 수정하지 마.
- ODsay 연동을 수정하지 마.
- Weather 연동을 수정하지 마.
- Supabase 저장/조회 로직을 수정하지 마.
- Kakao Map SDK를 수정하지 마.
- RecentPlaceList localStorage 로직을 수정하지 마.
- 디자인 변경하지 마.
- 레이아웃 변경하지 마.
- 패키지 설치하지 마.
- 대규모 리팩토링하지 마.
- 클라이언트에 ANTHROPIC_API_KEY를 노출하지 마.
- NEXT_PUBLIC_ANTHROPIC_API_KEY를 만들지 마.

서비스 표현 원칙:
- 위험도 점수는 실제 사고 확률이 아니라 “운전 위험 지수”다.
- “사고 확률”, “예측 확률”, “probability” 표현은 새로 추가하지 않는다.
- AFC 혼잡도는 실시간이 아니라 “과거 패턴 기반 예측형 혼잡도”로 표현한다.
- fallback은 운영 안정성을 위한 대체 응답으로 유지한다.
- 면허 반납은 강요가 아니라 가족과 함께 논의할 수 있는 의사결정 보조 관점으로 표현한다.
- 리포트는 고령자와 가족이 함께 판단할 수 있는 부드러운 문체를 유지한다.

점검 및 수정 지침:

1. 환경변수 확인
다음 환경변수가 서버에서 정상적으로 읽히는지 확인한다.
- ANTHROPIC_API_KEY

확인할 것:
- process.env.ANTHROPIC_API_KEY를 서버 전용 파일에서만 읽는가?
- components, client component, 브라우저 코드에서 참조하지 않는가?
- /api/health에서 키 값이 아니라 boolean만 노출되는가?
- 키가 없을 때 Claude 호출을 시도하지 않고 template fallback으로 안전하게 이동하는가?

주의:
- ANTHROPIC_API_KEY를 console.log하지 않는다.
- API 응답에 키 값을 포함하지 않는다.
- NEXT_PUBLIC_ANTHROPIC_API_KEY를 만들지 않는다.

2. generateClaudeReport 동작 확인
lib/report/generateClaudeReport.ts 또는 유사 파일을 확인한다.

확인할 것:
- Claude API endpoint 또는 SDK 호출이 현재 Anthropic API 형식에 맞는가?
- timeout이 설정되어 있는가? 예: 15초
- 호출 실패 시 generateTemplateReport로 fallback하는가?
- 응답 본문이 비어 있거나 형식이 예상과 다를 때 안전하게 fallback하는가?
- 리포트 결과에 generatedBy 또는 source 같은 필드가 있다면 "CLAUDE"와 "TEMPLATE"를 구분할 수 있는가?
- Claude 응답에 금지 표현이 포함될 가능성에 대비해 후처리 또는 fallback 검증이 있는가?

권장 fallback reason:
- ANTHROPIC_API_KEY_MISSING
- CLAUDE_HTTP_ERROR_STATUS_401
- CLAUDE_HTTP_ERROR_STATUS_403
- CLAUDE_HTTP_ERROR_STATUS_429
- CLAUDE_HTTP_ERROR_STATUS_500
- CLAUDE_TIMEOUT
- CLAUDE_EMPTY_RESPONSE
- CLAUDE_PARSE_ERROR
- CLAUDE_FETCH_FAILED
- CLAUDE_CONTENT_POLICY_OR_INVALID_TEXT
- UNKNOWN_CLAUDE_ERROR

3. /api/report route 확인
app/api/report/route.ts가 있다면 다음을 확인한다.

권장 응답 구조:
{
  "ok": true,
  "data": ReportContent,
  "meta": {
    "source": "CLAUDE" | "TEMPLATE",
    "fallback": boolean,
    "reason": string | null
  }
}

실제 Claude 성공 시:
- meta.source = "CLAUDE"
- meta.fallback = false
- reason = null 또는 "CLAUDE_SUCCESS"

fallback 시:
- meta.source = "TEMPLATE"
- meta.fallback = true
- reason = 실제 원인

주의:
- 키 누락, quota 초과, timeout, parse 실패를 구분할 수 있게 한다.
- 내부 에러 전문을 사용자 응답에 그대로 노출하지 않는다.
- 리포트 내용은 면허 반납을 강요하지 않는다.

4. /api/analyze 내 Claude 호출 확인
/api/analyze가 전체 분석 플로우에서 Claude 리포트를 호출한다면 다음을 확인한다.

확인할 것:
- ODsay/Weather 결과와 운전 위험 지수를 바탕으로 Claude 리포트를 생성하는가?
- Claude 실패 시 분석 전체가 실패하지 않고 템플릿 리포트로 대체되는가?
- fallbackFlags.report 또는 유사 필드로 fallback 여부가 표시되는가?
- Supabase 저장 시 Claude 또는 template 리포트 결과가 저장되는가?
- 응답에 ANTHROPIC_API_KEY나 내부 에러 전문이 포함되지 않는가?

5. 리포트 문구 검증
Claude 또는 template 리포트 모두 아래 원칙을 지켜야 한다.

필수 포함 또는 유지:
- “운전 위험 지수”
- “의사결정 보조”
- 가족과 함께 논의할 수 있다는 톤
- 대중교통 대안 검토 문구

금지:
- 사고 확률
- 예측 확률
- probability
- 반드시 반납
- 운전 금지
- 공포 조장 문구
- 실시간 혼잡도

주의:
- Claude 응답은 예측 불가능하므로 금지 표현 검증 또는 안전한 system/user prompt가 필요하다.
- 금지 표현이 감지되면 template fallback으로 대체하는 방식도 가능하다.
- 과도한 구현은 피하고 현재 구조에 맞는 최소 보강만 한다.

6. 안전한 서버 로그 추가
운영 확인을 위해 로그는 가능하되 민감정보는 제외한다.

권장 로그:
- Claude 호출 성공/실패 여부
- HTTP status
- reason code
- fallback 사용 여부
- elapsed time

금지 로그:
- ANTHROPIC_API_KEY 전체 값
- 전체 request body에 민감정보가 있을 경우 무분별한 출력
- Claude 원본 응답 전체를 과도하게 출력

7. 테스트 작성/보강
실제 Claude API를 테스트에서 호출하지 않는다.
fetch mock 또는 기존 abstraction mock을 사용한다.

필수 테스트:
- ANTHROPIC_API_KEY가 없으면 template fallback으로 응답한다.
- Claude 성공 응답을 ReportContent로 normalize한다.
- Claude HTTP 401/403/429/500 응답 시 reason을 구분한다.
- Claude timeout 또는 fetch 실패 시 template fallback으로 응답한다.
- Claude 빈 응답 또는 parse 실패 시 template fallback으로 응답한다.
- 금지 표현이 포함된 Claude 응답은 사용하지 않거나 안전하게 처리한다.
- API key가 응답 문자열에 포함되지 않는다.
- 리포트 문구가 면허 반납을 강요하지 않는다.

금지 표현:
- 시연용
- Mock
- MOCK
- 더미
- 사고 확률
- 예측 확률
- probability
- 실시간 혼잡도
- 반드시 반납
- 운전 금지

주의:
- 테스트명에 Mock/시연용/더미 표현을 새로 추가하지 않는다.
- 기존 테스트가 generatedBy 또는 source 값을 기대한다면 새 구조와 맞춘다.
